# gateway

Модуль API Gateway — единая точка входа (маршрутизатор) для всей экосистемы приложения.

## Назначение модуля
Этот сервис принимает все входящие запросы от клиентских приложений (мобильного фронтенда или веб-версии), проверяет их безопасность, авторизует пользователей и перенаправляет запросы в соответствующие внутренние сервисы (`core-backend` или `streaming-service`). 

Он изолирует внутреннюю архитектуру проекта от внешнего мира, обеспечивая безопасность и высокую скорость работы.

## Основной функционал
* **Умная маршрутизация (Routing):** Перенаправление запросов к каталогу (фильмы, категории) на `core-backend` и запросов тяжелого медиа-трафика (видео-клипы) на `streaming-service`.
* **Авторизация и аутентификация:** Проверка валидности пользовательских сессий и JWT-токенов перед тем, как пропустить запрос к логике приложения.
* **Ограничение частоты запросов (Rate Limiting):** Защита бэкенд-модулей от перегрузок, DDoS-атак и парсинга контента.
* **Централизованное логирование:** Сбор базовой аналитики по входящим запросам и времени их обработки для контроля стабильности системы.

## Технологический стек
* **Язык разработки:** Golang
* **Менеджер зависимостей:** Инициализирован через Go Modules (`go.mod`)
* **Протоколы взаимодействия:** HTTP/REST
